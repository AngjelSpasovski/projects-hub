# Internationalization Rules

- Every visible UI label, message, empty state, error, tooltip, and modal title uses a translation key.
- English keys live in `src/assets/i18n/en.json`; Macedonian keys live in `src/assets/i18n/mk.json` and use Cyrillic.
- New components must add both language values in the same change.
- Technical tags and proper product names can remain unchanged when translation would be misleading.
- `src/assets/i18n/i18n.spec.ts` verifies matching key sets, non-empty values, and project registry coverage.

Run `npm run test -- --watch=false` before release. A missing or blank translation must fail the test suite.
